# Release Notes Checklist

## Feature Section

- [ ] Heading format: ### \`feature.name\` [#PR](url)
- [ ] Introduction explains what and why
- [ ] Code example is practical and runnable
- [ ] Links to full documentation
- [ ] Mentions related features if applicable

## Writing Style

- [ ] Leads with user value, not technical details
- [ ] Uses active voice
- [ ] No typos or grammatical errors
- [ ] Proper punctuation (especially commas)
- [ ] Uses 1. for all numbered lists

## External Contributors

- [ ] If applicable, thanked with @username
- [ ] Check PR commits against team list in [references/k6-team-members.md](references/k6-team-members.md)

## Code Examples

- [ ] Do NOT require `<!-- md-k6:skip -->` markers in release notes
- [ ] Should be validated when tested
